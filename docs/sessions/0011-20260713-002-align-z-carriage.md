## The decision

**Z carriage rides the mode-in-shape rule (ADR-0002) as an axis kind; admission is a request-side
per-axis gate; declarations are facts.** In full (encoding settled with the implementation plan):

- **Declarations are native facts, never widened**: temperature = sample @ `2 m`; wind = samples at
  levels; cloud cover = the served cells `{[0,2 km], [2,6 km], [6,TOA], [0,TOA]}`; Open-Meteo X/Y =
  global (a real vendor commitment — server-side interpolation). Phase C's `_NEAR_SURFACE_Z = [0,10]`
  fat capability footprint is **interpretation polluting the fact slot** and is replaced at 002.
- **Axis encoding**: a base **`IntervalAxis`** — a single-cell `EnumerableAxis` defined by an
  `interval` (`extent == bounds == interval`, default containment `matches`) — is the enumerable span
  cell: the native cloud column `[0,TOA]` (no `RegularAxis` form, no `step=inf`/`nan`). **`VantageAxis`
  `(IntervalAxis)`** adds only the intersection `matches` override. `ContinuousAxis` leaves the Z story
  (X/Y footprint reach only). `RegularDomain` widens to **`GridDomain`** (`Mapping[AxisName,
  EnumerableAxis]`: `RegularAxis` X/Y/T; on Z a `VantageAxis`, a count-1 `RegularAxis` sample, or an
  `IntervalAxis` column).
- **Request Z mode = an axis kind**: a **`VantageAxis`** = vantage — the edge-authored acceptance
  window (`[0, ~10 m]`); a count-1 `RegularAxis` (level) or `IntervalAxis` (layer) = exact addressing
  (003's aliases `temperature_2m`, `cloud_cover_low`, `soil_temperature_6cm`). By closed projection the
  `VantageAxis` rides onto the served Coverage (`resample` sets `domain = selection.domain`) — never in
  a footprint; on the declared side only its `.extent` is read, so its inverted predicate never leaks.
- **Admission is a request-side per-axis gate** — `requested.matches(declared)` (dispatch on the
  *request* axis; the implementation-plan refinement of the earlier `declared.matches`). Default axis
  = `declared.extent.contains(self.extent)` (request inside footprint); **`VantageAxis` =
  `Interval.intersects(declared.extent)`** (overlap). **One predicate**: against a point sample
  `intersects` *is* membership (`2 ∈ [0,10]`), against a column *is* inclusion (`[0,10] ⊆ [0,TOA]`) —
  subsuming the quantifier reasoning with no per-declaration branch. Precedent: `RollingAxis`
  clock-window matching; the same cell arithmetic is the store's held-cell report (ADR-0006).
- **Admission ≠ selection** — the gate is liberal ("could this contribute?"); *which* admitted cell
  answers is the **maximal-served-cell** (statistics: the served cell containing the others; none →
  omission) or the **resampler** (samples: one representative). v1 exercises the trivial case only, so
  selection is **named, not built**; the bounded-layer divergence stays open
  ([concern #13](../concerns.md#13-candidate-admission-containment-vs-intersection)). Vantage-
  resolvability is **emergent from capability** — no `requestable-in-vantage` flag.
- **Resampler/Calculator boundary**: *resampler = same value, new geometry, no assumptions* (entailed
  by scale × statistic × extent_scaling); *Calculator = new value, declared assumptions, synthetic
  provenance*. Multi-level samples in a window → resampler representative (linear via u/v). Cloud
  layers → total needs an overlap assumption → **Calculator** (`cloud_cover @ [0,TOA]`, lineage
  low/mid/high) — the same gap-filling move as between-levels wind interpolation.
- **Closed projection intact**: response rides `sel.domain` (Z cell = the window, offsets absorbed
  into `bounds` per ADR-0002); native cells stay in the native record, recoverable via `SourceKey`.
  `Selection` stays one-domain; the session 0009 decline of per-parameter domains stands.
- **Vertical references**: confirmed already-decided (ADR-0002 — one `vertical_reference` per Z axis
  per Domain ⇒ single-reference requests by construction; cross-frame = Calculator). A proposed
  `ReferenceAxis` was withdrawn as solving a solved problem.

## Rejection trail (the structural loads)

1. **Fat union request** — the caller's domain conflated with native geometry; a column parameter
   inflates the union and breaks containment for narrow parameters.
2. **Fat capability reach + point request ("B2")** — consumer interpretation in the fact slot
   (indicts shipped `_NEAR_SURFACE_Z`).
3. **Precise capabilities + Z-intersection admission ("B3")** — per-axis special-casing of the
   predicate.
4. **Per-parameter Z cells in Selection ("B1-lite")** — breaks closed projection
   (`project(selection)` must return `sel.domain`).
5. **Kernel-dilated footprints** — kernel policy is not provider footprint; tolerance is request
   semantics (the vantage window *is* the consumer's tolerance).

## Validation

Stress-tested against a heterogeneous zoo: multi-level wind (resampler), cloud layers as **domain
declarations, not identities** (one functional, four cells; names are edge aliases), gusts/precip T
statistics (existing machinery), soil temperature (honestly excluded by a `[0,10]` window),
`pressure_msl`/isobaric (separate-frame requests per ADR-0002), station providers (X/Y *sample* axes —
the family generalizes horizontally). Much of the data-side design was **rediscovered** in ADR-0002's
existing text (fat cell absorbs offsets; vertical homogenization; coarsening follows the statistic);
the genuinely new half is the **matching** side, landed in ADR-0004.

## Remaining 002 align questions — all resolved this session

- **Internal-only carriage** → the **edge exposure table** (option c): a name exists ⇔ requestable;
  `wind_u`/`wind_v` simply have no entry. Default set / narration / validation = exposure ∩ woven
  capability. No `requestable` flag on `ParameterDef` — the canonical table stays facts; 003 extends
  the same table with alias desugaring.
- **Vendor mapping shape** → **`Tap`** (session 0002's `Channel`, renamed — a fitting on the vendor
  stream drawing off one conditioned parameter). **Single-output** (`produces: ParameterId`); wind =
  two taps over the same two vendor vars. `VendorVar { name, unit }` (`native_unit`→`unit`).
  `decode(arrays) → (values, present)` does quantity **transform** only. The tap table is the single
  leaf source for fetch vars, normalizer, and capability. Footprint assembly stays **leaf-private**
  (ADR-0004); the axis-kind slots (X/Y+T provider-level, Z per-tap) generalize to grid/station via
  the two-arm vertical rule (Z degenerate cell **or** a shared field axis for volumetric providers).
- **Units** → **verify-always, convert-on-ingest, no request knob**. Native→canonical conversion is
  the Normalizer's job (per tap, driven by `VendorVar.unit`); URL never negotiates units; wind
  `km/h→m/s` inline at 002. Shared conversion **catalogue** deferred to
  [ticket 010](../tickets/010-unit-conversion-edge.md) (forced by 004). Canonical units recorded in
  [ticket 002](../tickets/002-core-5-parameters.md) / [parameters.md](../parameters.md).
- **Precipitation variable** → Open-Meteo **total** `precipitation` (liquid-equivalent `mm`);
  rain/snow/showers decomposition is post-v1.
