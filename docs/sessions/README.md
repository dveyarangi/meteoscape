# Development sessions

This directory contains dated planning, alignment, and implementation records. Each file is a
historical snapshot: statements such as "not implemented," "next," or "lands in ticket 002" describe
the project at the time of that session and are not maintained as current status.

For the current implementation state and delivery order, see the
[v1 delivery status](../tickets/README.md). For durable decisions, follow the session's links to the
[architecture](../architecture.md), [ADRs](../adr), and active tickets.

Do not rewrite old sessions merely to reflect later delivery. Add a new session or update the owning
contract/ticket instead.
